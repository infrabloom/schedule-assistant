# Exemplar deliverables

Sanitized example deliverables contributed back from completed data center
schedule projects — a clean PM briefing, a critical-path analysis, a fragnet,
and so on. They show what "good" looks like for each deliverable type, so a
future build has a model to follow.

This folder starts empty. A project adds an exemplar only when it produced an
unusually clean deliverable worth reusing (see `MAINTENANCE.md`, Step 5).
Sanitize an exemplar the same way as a template XER: strip client and contractor
names, real dates, and dollar amounts before adding it here.

Naming: `[deliverable-type].md` — e.g. `pm-briefing.md`,
`critical-path-analysis.md`.
